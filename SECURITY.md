# Security Policy

## Reporting a Vulnerability

Please report security issues privately to the maintainers. Do not file public issues for vulnerabilities, credential leaks, auth bypasses, or data exposure bugs.

When reporting, include:

- a short description of the issue
- affected endpoints or components
- reproduction steps or a proof of concept
- impact assessment if known
- any suggested mitigations

## Response Expectations

We aim to:

- acknowledge reports promptly
- validate and triage severity
- coordinate a fix and release plan
- credit reporters when appropriate and desired

## Scope

Security issues may include:

- authentication or authorization bypass
- key or token leakage
- tenant data isolation failures
- prompt or request data exposure
- supply-chain or dependency risks with meaningful impact

## Supported Versions

Until a public release process is established, security fixes are handled on the actively maintained branch only.
